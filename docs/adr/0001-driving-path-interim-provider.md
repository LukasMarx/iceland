# DrivingPath provider: interim OpenRouteService behind a swappable adapter, Valhalla deferred

The concept doc names Valhalla as the target routing engine, but the Valhalla POC (Docker + Iceland OSM extract + F-road/closure costing) is unbuilt and flagged as the doc's biggest open risk. We need a route-calculating endpoint now. We'll build a `RoutingProvider` adapter interface (as the concept doc §5.2 prescribes) and implement it against hosted OpenRouteService for real car geometry + drive times immediately, persisting results in `DrivingPathCache` (renamed from `RoutingLegCache`). F-road/4WD/closure awareness is deferred to the Valhalla phase and surfaces via the `warnings` field without changing the contract. Swapping to Valhalla later is a one-file adapter change; the cache key already carries `vehicle` so 4WD entries re-route automatically on swap.
