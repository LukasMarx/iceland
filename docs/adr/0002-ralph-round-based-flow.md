# Ralph loop: round-based flow with strict gate↔review cycling, hybrid continuity, separate reviewer agent

The attempt-based flow (MAX_ATTEMPTS_PER_ISSUE=2, each attempt a fresh cold-start opencode invocation) was unreliable: session IDs were never captured (inheritStdio emptied stdout, so parseSessionIdFromTranscript always returned null), retries lost all prior context, e2e in the gate caused unfixable failures, and the 2-attempt cap gave up too early. We're replacing attempts with a single attempt per issue containing up to T=10 rounds, organized as a strict state machine: Implementation → Gate-fix loop (G=3) → Code-review → Review-fix loop (R=2), where every review-fix re-proves the gate and re-earns review sign-off. Continuity is hybrid: session-resume first, git-state fallback (cold start with `git diff main...branch` + a round log) when the session is missing or a round produces no new commits. Code-review is performed by a separate reviewer agent (cold-started, no shared session) that emits `REVIEW_CLEAN` or `REVIEW_FINDINGS:` with a numbered list. The gate drops e2e (agent can't fix it) and matches `ralph.md`'s project list exactly. Crash and no-commits count as rounds; double-failure (session-resume no-commits then fallback no-commits) abandons immediately. F3 infra failures retry then abandon, never return 0. Per-round timeouts are tiered: implementation 30m, fix rounds 20m, review 15m.

Rejected: session-only continuity (fragile across restarts); loose state machine (trusts review-fixes without re-reviewing); self-review by the implementer (blind spots); per-episode caps (allows cycling); flat 30m timeout (stuck fix rounds burn 30m).
